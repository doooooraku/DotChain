## 1. プロダクト概要

### 1-1. プロダクト名

- プロダクト名：**DotChain（ドットチェイン）**
- カテゴリ：習慣トラッカー / ライフログ系アプリ
- 対応プラットフォーム：**iOS / Android**（React Native + Expo）

### 1-2. 一言コンセプト

> **「1タップで“続ける”を可視化する、チェーン型習慣トラッカー」**

### 1-3. エレベーターピッチ（30秒説明）

> DotChain は、毎日の習慣を「1タップで記録」して、
> カレンダー上に**ドット（点）のチェーン**として可視化する習慣トラッカーです。
>
> 無料版でも3つの習慣をシンプルに続けられ、
> Pro プランでは習慣数の上限なし・広告なし・全カラーテーマ解放で、
> 自分らしい見た目とリズムで続けられます。
>
> 18言語対応で、世界中のユーザーが同じ体験を共有できます。

### 1-4. 提供価値（バリュープロポジション）

DotChain がユーザーに提供する価値を、具体的なシーンで定義します。

1. **「続けた証拠」が一目で分かる**
   - カレンダー上の「ドットのつながり（チェーン）」で、
     どれだけ続けたかが瞬時に分かる。

2. **1タップで記録が終わる“軽さ”**
   - アプリを開いて、タップして閉じるだけ。
   - 入力項目や文字入力を極限まで減らし、「めんどくささ」を排除する。

3. **視覚的なご褒美（UI演出）**
   - チェーンが伸びたときの小さなアニメーションや色の変化で、
     「今日もできた！」という達成感を演出する。

4. **無料でも十分試せる / Pro でしっかり使える**
   - Free：習慣3つまで + 広告あり + ダークテーマ固定
   - Pro：習慣無制限 + 広告なし + 全テーマ解放
   - まずは無料で体験 → 続けたい人が自然に Pro に上がる設計。

## 2. ターゲットユーザー / ペルソナ

### 2-1. メインターゲット

**年齢層**

- 主：20〜39歳
- 副：高校生〜大学生（16〜24歳）

**属性のイメージ**

- 勉強・筋トレ・読書・語学など「自己投資」系の習慣をつけたい人
- すでに日々のタスク管理アプリは使っているが、
  「習慣」だけを別でシンプルに管理したい人
- カジュアルゲーマー / ガジェット好き / デザインにこだわりがある人

### 2-2. 代表ペルソナ例

#### ペルソナA：社会人エンジニア（27歳）

- 平日は本業 + 副業開発で忙しい
- 毎日やりたいこと：
  - 英語の勉強
  - 筋トレ
  - 個人開発
- 要求：
  - 入力は極力シンプルにしたい（チェックボックス程度）
  - グラフよりも、「どれだけ途切れずに続いたか」を見たい
  - 夜寝る前に 30秒で入力を終わらせたい

#### ペルソナB：大学生（20歳）

- 資格勉強・読書・ランニングなどを習慣化したい
- スマホのデザイン・色合いにこだわりがある
- SNSに画面を載せることもある
- 要求：
  - 見た目がカッコいい / かわいいアプリを使いたい
  - 無料で始めたいが、「続けられたら課金もあり」くらいの温度感

### 2-3. 対象外とするユーザー（やらないこと）

- チームで習慣を共有・競争したい人（フレンド機能やランキングは v1.0 ではやらない）
- 複雑なタスク管理やプロジェクト管理を求める人
- カロリー・体重などの詳細なヘルスデータ連携を求める人

→ DotChain v1.0 は、**「自分ひとりで淡々と続ける人」向けのミニマルな習慣アプリ**にフォーカスします。

## 3. 解決したい課題

### 3-1. 現状の課題

1. **習慣アプリは機能が多すぎて、逆に続かない**
   - カテゴリ・タグ・ノート・リマインダーなど、入力項目が多くて続かない。

2. **毎日続けた「結果」が分かりづらい**
   - 細かいグラフや数字はあるが、「結局どれだけ続いたの？」が直感で分からない。

3. **無料版が制限だらけ / 広告が重すぎる**
   - 無料だとほとんど使いものにならず、最初から課金前提になっているアプリも多い。

### 3-2. DotChain が解決するポイント

1. **1タップ / 1画面で完結**
   - 記録操作は「今日やったかどうか」のみ。
   - 1つの画面で「今日の習慣一覧 + カレンダー」を完結させる。

2. **チェーンで“続けた量”を見せる**
   - カレンダーのドットが途切れずつながることで、「やめたくない」心理を刺激する。

3. **無料版でも“きちんと使える”ラインを明確に**
   - Free でも 3習慣まで「本気で」続けられる。
   - 「4つ目が欲しくなったとき」が Pro への自然なトリガーになる。

## 4. プロダクトゴール & KPI

### 4-1. 期間ごとの目標（定量）

**ローンチ 12ヶ月以内（v1.0〜v1.x）**

- 累計ダウンロード数：**50,000** 以上
- 月間アクティブユーザー（MAU）：**5,000** 以上
- Pro サブスクリプション会員数：**300** 以上
- 有料転換率（Free → Pro）：**3〜5%**
- デイリー継続率：
  - D1 Retention：**40%以上**
  - D7 Retention：**20%以上**
  - D30 Retention：**10%以上**

**3年以内の中期目標**

- 累計ダウンロード数：**300,000** 以上
- MAU：**30,000** 以上
- Pro 会員数：**3,000〜5,000**
- サブスク + 広告の月間売上：**50〜100万円** レベル

※これらの数値はあくまで「方向性」であり、
   1_基本仕様書 / 5_テスト仕様書 で追うべき指標のベースとします。

### 4-2. 主要KPIの定義

ここで出てくる用語を、初心者にも分かるように定義します。

- **MAU（Monthly Active Users）**
  - 直訳：月間アクティブユーザー数
  - 意味：過去30日以内にアプリを1回以上起動したユーザー数
  - イメージ：
    - 1ヶ月の間で「1回でも使った人」が何人いるか

- **Retention（リテンション / 継続率）**
  - 例：D1, D7, D30
  - D1：インストールした翌日に、何％の人がもう一度アプリを開いたか
  - D7：インストールから7日目に、何％の人が戻ってきたか

- **有料転換率（Conversion Rate）**
  - 無料ユーザーのうち、何％が Pro に課金したか
  - 例：
    - 1000人の無料ユーザー → 50人が Pro 課金
    - 有料転換率 = 50 ÷ 1000 × 100 = **5%**

## 5. 競合・参考アプリとポジショニング

### 5-1. 代表的な競合 / 参考アプリ

- **Loop Habit Tracker（Android）**  
  シンプルな無料 + オープンソース習慣トラッカー。
  - 公式サイト：https://loophabits.org/  
  - Play Store：https://play.google.com/store/apps/details?id=org.isoron.uhabits  

- **Streaks（iOS / macOS）**  
  Apple Design Award 受賞の習慣アプリ。連続日数（ストリーク）に特化。  
  - 公式サイト：https://streaksapp.com/  
  - App Store：https://apps.apple.com/app/streaks/id963034692

- **Habitify**  
  マルチプラットフォーム対応の高機能習慣アプリ。  
  - 公式サイト：https://habitify.me/  
  - ダウンロードページ：https://habitify.me/download

- **Productive**  
  「ポジティブな習慣づくり」にフォーカスした習慣トラッカー。  
  - 公式サイト：https://productiveapp.io/  

これらを参考にしつつ、DotChain のポジションを明確にします。

### 5-2. DotChain のポジショニング

DotChain は、上記競合と比較して次のような立ち位置を狙います。

1. **UIの一貫した世界観（ネオンサイバー系）**
   - Loop が「ミニマル & マテリアルデザイン」なのに対して、
   - DotChain は「ネオンサイバーな色合い＋シンプルUI」で、
     見た瞬間に世界観が伝わるデザインを目指す。

2. **“チェーン”の視覚化に全振り**
   - 連続日数（ストリーク）を一番の主役に置く。
   - Streaks に近いが、より「ドットがつながる」というビジュアルを強調。

3. **Freeで3習慣まで本気で使える**
   - 多くのアプリは Free でできることが制限されがちだが、
     DotChain は **3習慣までなら本気運用可能** な設計にする。

4. **多言語対応（18言語）で世界展開を前提**
   - Habitify などと同じく多言語支援を強め、最初からグローバルを見据えた設計。

## 6. v1.0 におけるコア機能と優先度

ここでは詳細仕様ではなく、**戦略レベルでの機能定義**と優先度を記載します。詳細は 1_基本仕様書・2_機能設計書に譲ります。

### 6-1. コア機能一覧（戦略レベル）

- **F-01：習慣の追加・編集・削除**
  - 目的：ユーザーが「続けたい行動」をアプリ内に登録できる状態にする。
  - v1.0：1日1回チェックするシンプルな2値（やった / やってない）のみ対応。
  - 優先度：**MUST**

- **F-02：習慣実行の記録（チェーン表示）**
  - 目的：毎日の実行をワンタップで記録し、カレンダー上にチェーンとして表示する。
  - v1.0：日単位のみ（時間単位・週単位はやらない）。
  - 優先度：**MUST**

- **F-03：Free / Pro のプラン分岐**
  - Free：
    - 習慣登録は **3つまで**
    - 広告表示あり（画面下バナー）
    - テーマは Dark のみ
  - Pro：
    - 習慣登録数 **無制限**
    - 広告なし
    - 全カラーテーマ解放（Neon Pink / Cyber Blue など）
  - 優先度：**MUST**

- **F-04：Proプラン購入（RevenueCat連携） & 復元**
  - 目的：サブスクリプション課金の実装。
  - 購入フロー：
    - Pro画面から「月額」「年額」のいずれかを購入。
    - 成功 → Pro権限付与・SecureStore保存。
    - 失敗 → エラーメッセージを Toast で表示。
  - 復元フロー：
    - 設定画面の「購入を復元」から、過去の購入を復元。
    - 成功時は Alert、失敗時・復元対象なしは Toast。
  - 優先度：**MUST**

- **F-05：多言語対応（18言語）**
  - 対応言語：英語・日本語を含む 18言語（アラビア語除外）。
  - v1.0：文言のみの切替でOK。レイアウトは LTR のみ（RTL対応は行わない）。
  - 優先度：**MUST**

- **F-06：テーマ切替（Free / Pro 連動）**
  - Free：Dark 固定。他テーマはロック表示。
  - Pro：Neon Pink / Cyber Blue などを選択可能。
  - 優先度：**MUST**

- **F-07：広告表示（AdMob バナー）**
  - Freeのみ表示。Proは完全に非表示。
  - 優先度：**MUST**

### 6-2. v1.0 ではやらないと決めた機能

- ウィジェット / ホームウィジェット連携（F-08）
- アプリアイコン変更（Pro限定アイコン）
- RTLレイアウト対応（右→左言語の反転）
- アプリ内の Open Source Licenses 一覧画面

これらは「将来やるかもしれない」ではなく、**v1.0〜v1.x のスコープ外**とします。

## 7. ビジネスモデル / 価格戦略

### 7-1. マネタイズの柱

DotChain の収益は、次の2本柱で構成します。

1. **サブスクリプション（Proプラン）**
   - 機能的価値：習慣数無制限 / 広告なし / 全テーマ解放
   - 心理的価値：「本気で習慣を続けるための投資」という位置付け

2. **広告（Freeユーザー向け バナー広告）**
   - 最小限の邪魔にならないバナーのみ（全画面広告は採用しない）
   - 「無料でもしっかり使えるが、Proにするとスッキリ」という体験を設計

### 7-2. 価格イメージ（例）

※実際の価格はストアのレート・競合状況を踏まえて最終決定する。ここでは戦略の方向性を示す。

- 月額：**¥360〜¥480** / 月
- 年額：**¥2,800〜¥3,600** / 年
  - 目安：月額の 7〜8ヶ月分程度に設定し、「年額の方がお得」と感じる差を付ける。

### 7-3. 単位経済のラフイメージ

- 目標：
  - 1 Proユーザーあたり、**平均 ¥300〜¥400 / 月** の売上（LTV）
  - 広告収入：Freeユーザー 1人あたり **¥10〜¥30 / 月** を目安に、過剰な広告は出さない

## 8. プラットフォームと技術スタック（戦略レベル）

この章では、詳細な実装ではなく「なぜその技術構成なのか」を記載します。具体的な技術仕様は 3_詳細設計書 にて扱います。

### 8-1. 技術スタック（大枠）

- クライアント：**React Native + Expo + TypeScript**
- UI：**Tamagui**（デザインシステム＋クロスプラットフォームUI）
- 状態管理：**Zustand**（習慣ストア / Proストア / 設定ストア）
- ストレージ：ローカルDB（例：SQLite / MMKV）
- 課金：**RevenueCat**
- 広告：**Google AdMob（react-native-google-mobile-ads）**
- セキュリティ：**SecureStore** による Proフラグ・匿名IDの保存

### 8-2. 開発者がよく使う代表的なコマンド（概要）

ここでは、開発を始めるときに頻出するコマンドと、その意味を簡単にまとめます。

> ※実際の詳細な手順・バージョン固定などは 4_実装ルール.md で扱います。

- `pnpm install`
  - 読み方：ピーエヌピーエム インストール
  - 意味：`package.json` に書かれたライブラリ（React Native, Expo, Tamagui など）を一括でインストールするコマンド。
  - イメージ：アプリが使う「部品」をすべてダウンロードしてくる作業。

- `pnpm expo start`
  - 読み方：ピーエヌピーエム エキスポ スタート
  - 意味：Expo の開発サーバを起動し、スマホやエミュレータでアプリを動かせる状態にするコマンド。
  - イメージ：ゲームでいう「デバッグモードで起動」ボタン。

- `pnpm test`
  - 読み方：ピーエヌピーエム テスト
  - 意味：自動テストをまとめて実行するコマンド。アプリの重要な機能が壊れていないかを確認する。
  - イメージ：出荷前チェックを自動でやるボタン。

## 9. 品質・UX戦略

### 9-1. 品質目標（定量）

- クラッシュ率：
  - 月次クラッシュ率（クラッシュセッション ÷ 全セッション） **1%未満**
- 体感速度：
  - コールドスタート（アプリ初回起動） **3秒以内**
  - 2回目以降の起動 **1.5秒以内**
- 不具合修正：
  - 重大バグ（クラッシュ / 課金関連）は発見から **72時間以内に修正リリース** を目標。

### 9-2. UX原則

- 「**1画面 = 1目的**」
  - ホーム画面は「今日の習慣をチェックする」ことに特化。
  - 設定画面は「テーマ / 言語 / Pro」など環境設定だけに集中。

- 「入力は最小限」
  - v1.0 ではメモや数値入力を行わない。

- 「常に視覚的なフィードバックを返す」
  - タップしたときに必ず色・サイズ・アニメーションの変化があること。

## 10. 開発・運用方針（高レベル）

### 10-1. リリース方針

- v1.0 では **「ミニマムだが気持ちよく動く」** ことを最優先。
- v1.1 以降で、ユーザーフィードバックを受けて小さな改善を繰り返す。
- 2週間〜1ヶ月単位で小さなアップデートを継続。

### 10-2. 将来拡張の方向性（参考）

※あくまで「長期的なアイデア」であり、v1.0 の仕様には含めません。

- 習慣ごとの詳細メモ / メモリプレイ
- リマインダー通知（時間指定）
- 友達と習慣を共有するソーシャル要素
- チャレンジモード（30日チャレンジ）

## 11. 用語集（初心者向け）

- **習慣トラッカー**
  - 毎日行う行動（例：勉強・筋トレ）を記録し、「続いているかどうか」を見える化するアプリ。

- **チェーン（Streak）**
  - 連続して習慣を達成できた日数。
  - 例：3日連続で筋トレ → ストリーク3日。

- **サブスクリプション**
  - 月額や年額で継続的に料金を支払う課金形式。

- **Free / Pro**
  - Free：無料版。機能に制限がある代わりに無料で使える。
  - Pro：有料版。機能が解放され、広告もなくなる。

- **バナー広告**
  - 画面の端に細長く表示される広告。動画広告や全画面広告に比べて控えめ。

- **多言語対応**
  - アプリの表示言語を複数用意して、ユーザーの言語に合わせて切り替えられるようにすること。

- **LTR / RTL**
  - LTR（Left To Right）：左から右へ読む言語（日本語・英語など）。
  - RTL（Right To Left）：右から左へ読む言語（アラビア語など）。DotChain v1.0 では LTR のみ対応。
